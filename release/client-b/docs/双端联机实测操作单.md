# 双端联机实测操作单

## 1. 目的

本文档用于完成当前项目最后一段关键验收：

- 第二台客户端接入验证
- 两台客户端 VPN 互通验证
- 目标游戏虚拟 IP 直连联机验证

完成这份操作单后，项目即可从“单端可用”推进到“联机能力已实测”。

## 2. 测试对象

### 客户端 A

- 使用配置：`output/client-configs/client-01.conf`
- 虚拟 IP：`10.99.0.11`
- 建议配置文件：`client-a.json`

### 客户端 B

- 使用配置：`output/client-configs/client-02.conf`
- 虚拟 IP：`10.99.0.12`
- 建议配置文件：`client-b.json`

### 服务端

- 公网地址：`47.119.136.200`
- VPN 地址：`10.99.0.1`
- 端口：`51820/udp`

## 3. 测试前准备

两台 Windows 机器都需要满足：

- 已安装 `WireGuard for Windows`
- 已拿到对应的 `.conf`
- 已拿到 `game-booster-client.exe`
- 已准备对应的 `client.json`
- 可使用管理员权限运行命令

### 3.1 安装 `WireGuard for Windows`

如果目标机器还没有安装 `WireGuard for Windows`，先在 PowerShell 中执行：

```powershell
winget install --id WireGuard.WireGuard -e --source winget --accept-package-agreements --accept-source-agreements
```

安装完成后检查：

```powershell
Test-Path "C:\Program Files\WireGuard\wireguard.exe"
```

预期：

- 返回 `True`

### 3.2 建议分别准备如下 JSON

#### 客户端 A：`client-a.json`

```json
{
  "serverEndpoint": "47.119.136.200:51820",
  "serverVPNIP": "10.99.0.1",
  "tunnelName": "client-01",
  "configPath": "client-01.conf"
}
```

#### 客户端 B：`client-b.json`

```json
{
  "serverEndpoint": "47.119.136.200:51820",
  "serverVPNIP": "10.99.0.1",
  "tunnelName": "client-02",
  "configPath": "client-02.conf"
}
```

## 5. 单机接入检查

### 客户端 A

1. 普通终端执行：

```powershell
.\game-booster-client.exe -config .\client-a.json status
```

预期：

- 返回 `未连接`

2. 管理员终端执行：

```powershell
.\game-booster-client.exe -config .\client-a.json connect
```

3. 再执行：

```powershell
.\game-booster-client.exe -config .\client-a.json status
```

预期：

- 返回 `已连接`

4. 执行：

```powershell
ping 10.99.0.1
```

预期：

- 能收到回复

### 客户端 B

重复客户端 A 的步骤，但使用：

- `client-b.json`
- `client-02.conf`

预期同样为：

- `status` 显示 `已连接`
- `ping 10.99.0.1` 成功

## 6. 双客户端互通验证

```

预期：

- 能收到回复

### 客户端 B ping 客户端 A

在客户端 B 上执行：

```

预期：

- 能收到回复

## 7. 游戏联机验证

### 适用前提

当前首期方案优先支持：

- 通过虚拟 IP 直连的游戏联机方式

当前不保证：

- 局域网广播自动发现房间

因此如果游戏支持手动输入主机 IP，优先使用该方式。

### 建议步骤

1. 让客户端 A 作为房主
2. 客户端 A 启动游戏并进入可联机场景
3. 客户端 B 启动游戏并选择“直连 IP”或等价方式
4. 在客户端 B 中输入房主虚拟 IP：`10.99.0.11`
5. 发起连接

### 预期结果

- 客户端 B 能成功进入客户端 A 的联机会话
- 联机过程可维持稳定
- 无明显的持续断线

## 8. 失败时的快速排查顺序

### 场景一：两台客户端都连不上服务端

先检查：

1. 服务端 `wg-quick@wg0` 是否为 `active`
2. 服务端是否监听 `51820/udp`
3. 云安全组是否放行 `51820/udp`
4. 客户端是否以管理员权限执行 `connect`
5. 目标机器是否已安装 `WireGuard for Windows`

### 场景二：能 ping `10.99.0.1`，但客户端之间互 ping 不通

先检查：

1. 服务端 `iptables -S FORWARD`
2. `wg0.conf` 是否保留 `PostUp/PostDown`
3. 服务端 peer 是否正确写成：
   - `10.99.0.11/32`
   - `10.99.0.12/32`
4. 两台客户端是否都已真实连接成功

### 场景三：客户端互 ping 通，但游戏无法发现房间

说明：

- 游戏可能依赖广播发现

处理方式：

- 尝试使用“直连 IP”方式
- 直接输入房主虚拟 IP `10.99.0.11`

## 9. 验收记录模板

建议记录如下结果：

- 客户端 A 是否成功连接：是 / 否
- 客户端 B 是否成功连接：是 / 否
- 客户端 A 是否可 ping `10.99.0.1`：是 / 否
- 客户端 B 是否可 ping `10.99.0.1`：是 / 否
- 客户端 A 是否可 ping `10.99.0.12`：是 / 否
- 客户端 B 是否可 ping `10.99.0.11`：是 / 否
- 游戏是否成功联机：是 / 否
- 如果失败，失败步骤位于：连接服务端 / 双端互通 / 游戏直连

## 10. 当前完成判定

满足以下条件即可判定项目核心目标完成：

- 两台客户端都能连接服务端
- 两台客户端之间互相 `ping` 通
- 至少一个目标游戏可以通过虚拟 IP 直连联机成功

如果只完成到“客户端都能接入服务端”，则项目属于：

- VPN 通道完成
- 联机能力基础完成
- 游戏实测验收未完成
