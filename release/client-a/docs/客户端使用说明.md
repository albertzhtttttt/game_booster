# 客户端使用说明

## 1. 适用范围

本文档适用于当前项目首期交付的最小 Windows 联机客户端。

当前客户端目标仅为：

- 连接到 `47.119.136.200` 上的 `WireGuard` 服务端
- 建立 VPN 联机通道
- 提供 `connect / disconnect / status` 三个最小命令

不包含 GUI、账号系统、自动更新、多节点切换等额外功能。

## 2. 交付物位置

当前客户端相关文件：

- 可执行文件：`output/build/game-booster-client.exe`
- 示例配置：`client.example.json`
- 第一台客户端配置：`output/client-configs/client-01.conf`
- 第二台客户端配置：`output/client-configs/client-02.conf`

## 3. 使用前置条件

在 Windows 客户端上使用前，需要满足以下条件：

- 已安装 `WireGuard for Windows`
- 以管理员权限运行客户端的 `connect` 和 `disconnect`
- 本地存在分配给该机器的 `.conf` 配置文件
- 云安全组或网络策略未阻止访问 `47.119.136.200:51820/udp`

## 4. 安装 `WireGuard for Windows`

如果目标机器还没有安装 `WireGuard for Windows`，推荐按下面步骤执行。

### 4.1 使用 `winget` 安装

在 PowerShell 中执行：

```powershell
winget install --id WireGuard.WireGuard -e --source winget --accept-package-agreements --accept-source-agreements
```

### 4.2 安装完成后检查

安装完成后，可在 PowerShell 中执行：

```powershell
Test-Path "C:\Program Files\WireGuard\wireguard.exe"
```

如果返回：

- `True`

说明安装成功。

也可以执行：

```powershell
Get-ChildItem "C:\Program Files\WireGuard\wireguard.exe"
```

### 4.3 如果 `winget` 不可用

如果目标机器没有 `winget`，则需要用户先自行安装 `WireGuard for Windows`，再继续执行后续步骤。

## 5. 配置文件说明

客户端通过 JSON 配置文件运行，示例见：`client.example.json`

示例内容：

```json
{
  "serverEndpoint": "47.119.136.200:51820",
  "serverVPNIP": "10.99.0.1",
  "tunnelName": "client-01",
  "configPath": "output/client-configs/client-01.conf"
}
```

字段说明：

- `serverEndpoint`：服务端公网地址，仅用于显示和校验
- `serverVPNIP`：服务端在 VPN 中的虚拟 IP，用于状态探测，当前为 `10.99.0.1`
- `tunnelName`：本机隧道名称，对应 Windows 服务名中的隧道标识
- `configPath`：WireGuard 配置文件路径
- `wireGuardPath`：可选字段，用于手动指定 `wireguard.exe` 路径

## 6. 命令用法

### 6.1 查看状态

```powershell
.\output\build\game-booster-client.exe -config .\client.example.json status
```

可能返回：

- `未连接`
- `连接中`
- `已连接`
- `连接失败`

### 6.2 建立连接

必须使用管理员权限运行：

```powershell
.\output\build\game-booster-client.exe -config .\client.example.json connect
```

如果不是管理员权限，当前版本会提示需要管理员权限。

### 6.3 断开连接

同样需要管理员权限：

```powershell
.\output\build\game-booster-client.exe -config .\client.example.json disconnect
```

## 7. 推荐使用方式

### 方式一：管理员 PowerShell

1. 用管理员身份打开 PowerShell
2. 进入项目目录
3. 执行 `connect`
4. 用 `status` 检查状态
5. 联机完成后执行 `disconnect`

### 方式二：普通终端查看状态，管理员终端连接/断开

- 普通终端可执行：`status`
- 管理员终端执行：`connect`、`disconnect`

## 8. 首次验证步骤

单客户端验证步骤：

1. 安装并确认 `WireGuard for Windows` 已就绪
2. 确认使用分配给本机的 `.conf` 文件
3. 先执行 `status`，预期返回 `未连接`
4. 以管理员权限执行 `connect`
5. 再执行 `status`，预期返回 `已连接`
6. 执行 `ping 10.99.0.1`
7. 联机结束后以管理员权限执行 `disconnect`
8. 再次执行 `status`，预期返回 `未连接`

## 9. 双客户端验证步骤

当两台 Windows 客户端都已拿到自己的配置后：

- 第一台使用 `client-01.conf`
- 第二台使用 `client-02.conf`

验证步骤：

1. 两台机器都连接 VPN
2. 第一台 `ping 10.99.0.12`
3. 第二台 `ping 10.99.0.11`
4. 如果互通成功，则可进行游戏虚拟 IP 直连联机

## 10. 常见问题

### 10.1 `Access is denied`

原因：

- 启动或关闭 `WireGuard` 隧道服务需要管理员权限

处理方式：

- 以管理员身份运行 PowerShell 或终端后重试

### 10.2 `未找到 wireguard.exe`

原因：

- 本机未安装 `WireGuard for Windows`
- 或安装路径不在默认位置

处理方式：

- 按本文档第 4 节先安装 `WireGuard for Windows`
- 或在 JSON 配置中增加 `wireGuardPath`

### 10.3 `已连接` 失败或 `连接失败`

可能原因：

- 服务端未运行
- 云安全组未放行 `51820/udp`
- 客户端配置文件错误
- 本机网络拦截 UDP

处理方式：

- 检查服务端状态
- 检查是否能访问 `47.119.136.200:51820`
- 检查配置文件是否与服务器 peer 对应
