# 渐进执行计划

基于 `docs/需求文档.md` 与 `docs/技术方案.md`，首期按“**先打通联机，再做最小客户端包装**”推进。

## 使用说明

- `[先做]`：必须先完成，否则后续任务无法推进。
- `[可并行]`：满足依赖后，可与同阶段其他任务并行推进。
- `[验证]`：用于确认当前阶段是否达成。

## 关键依赖关系总览

- `Phase 0 -> Phase 1 -> Phase 2 -> Phase 3 -> Phase 4 -> Phase 5`
- 服务端必须先打通，否则客户端开发无法完成真实联调。
- 客户端配置模板必须先固定，否则 `connect` / `status` 逻辑无法稳定实现。
- 双客户端联调必须在 CLI 主流程完成后进行。

## 最小里程碑

- [x] `M1`：服务端 `WireGuard` 成功运行
- [x] `M2`：至少一台客户端可连通服务端
- [x] `M3`：最小 CLI 客户端可执行 `connect / disconnect / status`
- [ ] `M4`：两台客户端可互通
- [ ] `M5`：目标游戏通过虚拟 IP 联机成功

## 当前推荐起手顺序

1. 先完成 `Phase 1`，打通服务端。
2. 再完成 `Phase 2`，拿到至少两份真实客户端配置。
3. 然后完成 `Phase 3`，做最小 CLI 客户端。
4. 最后执行 `Phase 4` 联机验证。
