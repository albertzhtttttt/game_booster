# 后续扩展候选项

## 1. 当前前提

这些内容都不属于当前首期最小可用范围。

只有在以下条件完成后，才建议考虑：

- 第二台客户端验证完成
- 双客户端互通完成
- 目标游戏联机验证完成

## 2. 第二阶段优先候选

### 2.1 简单 GUI 包装

目标：

- 用最小桌面界面替代命令行
- 提供连接、断开、状态显示按钮

价值：

- 降低普通用户使用门槛

### 2.2 自动导入配置

目标：

- 自动读取并导入 `.conf`
- 减少手工配置步骤

价值：

- 降低配置错误率

### 2.3 第二台客户端快速发放工具

目标：

- 生成新客户端配置
- 自动追加服务端 peer

价值：

- 降低新增用户接入成本

## 3. 第三阶段候选

### 3.1 网络诊断面板

目标：

- 显示是否安装 WireGuard
- 显示隧道状态
- 显示到 `10.99.0.1` 的 ping 状态

### 3.2 日志输出

目标：

- 将连接、断开、状态检测结果写入本地日志文件

### 3.3 一键管理员提权

目标：

- 在 Windows 上自动以管理员方式重启自身执行 `connect` / `disconnect`

## 4. 暂不建议过早做的内容

以下内容复杂度更高，不建议在首期验证未闭环前投入：

- 广播找房支持
- 二层局域网模拟
- 多节点切换
- 智能选路
- 用户系统
- 后台控制面
- 配置签发 API
- 流量统计与订阅
- 自动更新

## 5. 推荐优先级

推荐顺序：

1. 完成双客户端和游戏实测
2. 做简单 GUI
3. 做自动导入配置
4. 做客户端配置生成工具
5. 再考虑更复杂的控制面和扩展功能
