## 5. 新增客户端流程

当前首期采用人工发放配置，不使用控制面系统。

新增客户端的最小流程：

1. 在服务器生成新的客户端密钥对
2. 分配新的虚拟 IP，例如 `10.99.0.13`
3. 生成新的 `.conf`
4. 将对应 peer 写入 `/etc/wireguard/wg0.conf`
5. 重启 `wg-quick@wg0`
6. 将对应 `.conf` 安全发给目标客户端

新增 peer 的配置格式：

```ini
[Peer]
PublicKey = <client-public-key>
AllowedIPs = 10.99.0.13/32
```

## 6. 当前重要注意事项

### 6.1 Docker 影响转发策略

当前服务器存在 Docker，且 `FORWARD` 默认策略为 `DROP``。

因此必须保留 `wg0.conf` 中的以下规则：

```ini
PostUp = iptables -I FORWARD 1 -i wg0 -j ACCEPT; iptables -I FORWARD 1 -o wg0 -j ACCEPT
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -D FORWARD -o wg0 -j ACCEPT
```

如果删除这些规则，客户端之间可能无法互通。

### 6.2 云安全组

除了服务器本机配置外，还必须确认云平台安全组已放行：

- `51820/udp`

否则客户端无法完成握手。

### 6.3 私钥保护

以下文件属于敏感文件：

- `/etc/wireguard/server_private.key`
- `/etc/wireguard/clients/*.key`
- `/etc/wireguard/clients/*.conf`

这些文件不能公开泄露。

## 7. 常见问题排查

### 7.1 客户端无法连接

优先检查：

- `wg-quick@wg0` 是否为 `active`
- `51820/udp` 是否在监听
- 安全组是否放行 `51820/udp`
- 客户端 `.conf` 是否与服务端 peer 对应

### 7.2 客户端连上但不能互通

优先检查：

- `iptables -S FORWARD` 中是否存在 `-i wg0 -j ACCEPT` 和 `-o wg0 -j ACCEPT`
- 客户端 `AllowedIPs` 是否正确
- 服务端 peer 的 `AllowedIPs` 是否按 `/32` 正确配置

### 7.3 `wg0` 启动失败

优先检查：

- `/etc/wireguard/wg0.conf` 格式是否正确
- 私钥是否存在且权限正确
- 端口是否被占用
