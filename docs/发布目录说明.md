# 最小发布目录说明

## 1. 目录用途

当前项目已生成两套可直接分发的最小发布目录：

- `release/client-a`
- `release/client-b`

它们分别对应两台 Windows 客户端。

## 2. 对应关系

### `release/client-a`

对应：

- 客户端 A
- 虚拟 IP：`10.99.0.11`
- 配置来源：`client-a.json` + `client-01.conf`

### `release/client-b`

对应：

- 客户端 B
- 虚拟 IP：`10.99.0.12`
- 配置来源：`client-b.json` + `client-02.conf`

## 3. 目录内容

每个目录都包含：

- `game-booster-client.exe`
- `client.json`
- `client.conf`
- `docs/客户端使用说明.md`
- `docs/双端联机实测操作单.md`
- `docs/最终验收记录表.md`
- `docs/两台机器执行命令清单.md`

## 4. 如何使用

### 发给第一台机器

把整个目录复制到第一台 Windows 机器：

- `release/client-a`

### 发给第二台机器

把整个目录复制到第二台 Windows 机器：

- `release/client-b`

## 5. 建议落地方式

在目标机器上建议放到固定目录，例如：

```text
C:\game-booster-test\
```

把目录里的所有文件原样保留。

## 6. 运行方式

在目标目录中：

### 查看状态

```powershell
.\game-booster-client.exe -config .\client.json status
```

### 建立连接

管理员终端执行：

```powershell
.\game-booster-client.exe -config .\client.json connect
```

### 断开连接

管理员终端执行：

```powershell
.\game-booster-client.exe -config .\client.json disconnect
```

## 7. 下一步

将两套目录分别发到两台机器后，直接按照：

- `docs/双端联机实测操作单.md`

执行双端互通与游戏联机实测即可。
