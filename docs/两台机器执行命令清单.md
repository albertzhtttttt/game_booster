# 两台机器执行命令清单

## 1. 使用说明

本文档用于在两台 Windows 机器上直接复制执行命令，完成：

- 安装 `WireGuard for Windows`
- 查看状态
- 建立连接
- 验证服务端连通
- 验证双端互通
- 断开连接

约定：

- 第一台机器使用 `release/client-a`
- 第二台机器使用 `release/client-b`
- 下文默认你已经把对应目录复制到目标机器，例如：`C:\game-booster-test\`

## 2. 两台机器都先执行：安装 `WireGuard for Windows`

如果目标机器还没有安装 `WireGuard for Windows`，先在 PowerShell 中执行：

```powershell
winget install --id WireGuard.WireGuard -e --source winget --accept-package-agreements --accept-source-agreements
```

安装完成后检查：

```powershell
Test-Path "C:\Program Files\WireGuard\wireguard.exe"
```

预期：

- 返回 `True`

## 3. 机器 A 执行命令

### 3.1 进入目录

```powershell
cd C:\game-booster-test
```

### 3.2 查看初始状态

```powershell
.\game-booster-client.exe -config .\client.json status
```

预期：

- `未连接`

### 3.3 管理员权限建立连接

```powershell
.\game-booster-client.exe -config .\client.json connect
```

### 3.4 再次查看状态

```powershell
.\game-booster-client.exe -config .\client.json status
```

预期：

- `已连接`

### 3.5 验证服务端连通

```powershell
ping 10.99.0.1
```

### 3.6 验证到机器 B 的连通

```powershell
ping 10.99.0.12
```

### 3.7 联机结束后断开

```powershell
.\game-booster-client.exe -config .\client.json disconnect
```

### 3.8 确认断开

```powershell
.\game-booster-client.exe -config .\client.json status
```

预期：

- `未连接`

## 5. 游戏联机时的关键输入

如果目标游戏支持直连 IP：

- 房主：机器 A
- 房主虚拟 IP：`10.99.0.11`
- 机器 B 直连地址：`10.99.0.11`

## 6. 服务端快速检查命令

如果联机失败，可以在当前机器上远程检查服务端：

### 6.1 查看服务端状态

```bash
ssh -o StrictHostKeyChecking=no root@47.119.136.200 "systemctl status wg-quick@wg0 --no-pager"
```

### 6.2 查看 WireGuard 状态

```bash
ssh -o StrictHostKeyChecking=no root@47.119.136.200 "wg show"
```

### 6.3 查看监听端口

```bash
ssh -o StrictHostKeyChecking=no root@47.119.136.200 "ss -lunp | grep 51820"
```

## 7. 推荐执行顺序

推荐顺序如下：

1. 两台机器都先安装 `WireGuard for Windows`
2. 机器 A `connect`
3. 机器 B `connect`
4. 两台机器都 `ping 10.99.0.1`
5. A `ping 10.99.0.12`
6. B `ping 10.99.0.11`
7. 再开始目标游戏直连测试
8. 测试完成后两边都 `disconnect`
