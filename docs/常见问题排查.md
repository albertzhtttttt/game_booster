# 常见问题排查清单

## 1. 客户端 `status` 返回 `未连接`

说明：

- 当前隧道服务不存在或未运行

处理步骤：

1. 确认是否已经执行过 `connect`
2. 确认 `connect` 是否以管理员权限执行
3. 确认 `.conf` 路径和 `tunnelName` 是否正确

## 2. `connect` 提示需要管理员权限

说明：

- 在 Windows 上安装或卸载 WireGuard 隧道服务需要管理员权限

处理步骤：

1. 用管理员 PowerShell 打开终端
2. 重新执行 `connect`

## 3. 提示未找到 `wireguard.exe`

说明：

- 本机未安装 `WireGuard for Windows`
- 或安装路径不在默认位置

处理步骤：

1. 安装 `WireGuard for Windows`
2. 如有必要，在 `client.json` 中添加 `wireGuardPath`

## 4. `status` 返回 `连接失败`

说明：

- 隧道服务存在，但到 `10.99.0.1` 的探测失败

处理步骤：

1. 执行 `ping 10.99.0.1`
2. 检查服务端 `wg show`
3. 检查安全组是否放行 `51820/udp`
4. 检查客户端配置与服务端 peer 是否匹配

## 5. 客户端能连服务端，但两台客户端不能互通

说明：

- 说明单端接入成功，但客户端之间转发链路仍有问题

处理步骤：

1. 检查服务端 `iptables -S FORWARD`
2. 确认 `wg0.conf` 中保留了 `PostUp/PostDown` 规则
3. 确认两个客户端的 `AllowedIPs` 和服务端 peer 的 `/32` 配置正确

## 6. 游戏无法自动发现房间

说明：

- 当前首期方案是三层 VPN 和虚拟 IP 直连，不保证局域网广播发现

处理步骤：

1. 优先使用游戏内“直连 IP”方式
2. 输入房主的虚拟 IP，例如 `10.99.0.11`

## 7. 连接后延迟高或不稳定

可能原因：

- 本机网络质量问题
- 服务端出口网络波动
- 云平台限速或线路质量差

处理步骤：

1. 先测试 `ping 10.99.0.1`
2. 再测试客户端之间互 ping
3. 区分是到服务端不稳，还是客户端互通不稳
